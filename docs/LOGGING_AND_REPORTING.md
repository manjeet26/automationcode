# Admin Panel Test Automation - Logging & Reporting Guide

## Overview

This test automation project now includes comprehensive logging and detailed test reporting capabilities. All tests are logged at multiple levels and comprehensive HTML reports are generated automatically.

## 📊 Features

### Logging Capabilities
- **Multi-level logging**: DEBUG, INFO, WARNING, ERROR, CRITICAL
- **Multiple handlers**:
  - Console output (INFO level and above)
  - File handlers (DEBUG level and above)
  - Test-specific log files
  - Centralized pytest log file
- **Detailed log information**: Timestamp, logger name, level, filename, line number, function name, and message

### Reporting Features
- **Pytest HTML Report**: Auto-generated with pytest-html plugin
- **Comprehensive Report**: Custom HTML report with:
  - Test execution summary
  - Individual test logs (expandable)
  - Pytest output logs
  - Test metadata (timestamp, environment, success rate)
  - Responsive design with dark theme for code blocks
  - Collapsible sections for easy navigation

## 🚀 Quick Start

### 1. Install Dependencies

```bash
pip install -r requirements.txt
```

### 2. Run Tests with Logging

#### Option A: Run tests directly with pytest
```bash
pytest tests/ -v --log-cli=true
```

#### Option B: Run tests with full report generation
```bash
python run-tests-with-report.py
```

#### Option C: Run specific tests
```bash
pytest tests/test_smoke.py -v --log-cli=true
pytest tests/test_login.py -v --log-cli=true
```

#### Option D: Run with specific markers
```bash
pytest -m smoke -v --log-cli=true
```

### 3. Generate Reports Manually

After tests complete, generate or regenerate reports:

```bash
python generate-comprehensive-report.py
```

## 📝 Logging Configuration

### Pytest Configuration (pytest.ini)

The `pytest.ini` file contains comprehensive logging configuration:

```ini
[pytest]
addopts = 
    --log-cli=true                          # Show logs in console
    --log-cli-level=INFO                    # Console log level
    --log-file=test-results/logs/pytest.log # File for pytest logs
    --log-file-level=DEBUG                  # File log level
    --tb=short                              # Short traceback format
    -v                                      # Verbose output
```

### Custom Logger Setup (utils/logger.py)

The `LoggerConfig` class provides:
- Automatic log directory creation
- Dual handlers (file and console)
- Detailed formatting with timestamps and context
- Test-specific log files

### Test Configuration (tests/conftest.py)

The conftest.py automatically:
- Creates log directories
- Sets up root logger for all tests
- Creates test-specific loggers
- Provides a `logger` fixture for all tests
- Logs test start and end events

## 💻 Using Logger in Tests

### Example: Using the logger fixture

```python
@pytest.mark.smoke
def test_smoke_admin_login_page_loads(page: Page, logger):
    logger.info("Test: Verify admin login page loads")
    open_admin_login(page, logger)
    
    logger.info("Verifying username input is visible")
    expect(page.locator(USERNAME_INPUT)).to_be_visible()
    logger.info("✅ Test passed")
```

### Log Levels

```python
logger.debug("Detailed diagnostic information")

```

## 🛠️ Customization

### Changing Log Directory
Edit `tests/conftest.py`:
```python
def pytest_runtest_setup(item):
    current_test_logger = LoggerConfig.setup_logging(item.name, log_dir="custom-logs")
```

### Changing Log Format
Edit `utils/logger.py`:
```python
detailed_formatter = logging.Formatter(
    'YOUR_CUSTOM_FORMAT_HERE',
    datefmt='%Y-%m-%d %H:%M:%S'
)
```

### Changing Report Template
Edit `generate-comprehensive-report.py` and modify the HTML template in the `generate_html_report()` function.

## 🚨 Troubleshooting

### No logs generated?
1. Ensure `test-results/logs` directory exists or will be created
2. Check file permissions
3. Verify pytest is using the correct `pytest.ini`

### Reports are empty?
1. Ensure tests have executed successfully
2. Check that log files exist in `test-results/logs/`
3. Run `python generate-comprehensive-report.py` manually

### Logger not available in test?
1. Ensure your test function has the `logger` fixture parameter
2. Check that conftest.py is in the tests directory
3. Verify conftest.py is properly configured

## 🎓 Best Practices

1. **Log Important Steps**: Log before important actions (navigation, login, assertions)
2. **Use Appropriate Levels**: 
   - INFO for test flow
   - DEBUG for detailed diagnostics
   - WARNING for potential issues
   - ERROR for failures
3. **Include Context**: Always provide meaningful context in log messages
4. **Review Logs Regularly**: Check logs to understand test behavior
5. **Keep Logs Clean**: Remove old log files periodically
