# Quick Reference Guide - Logging & Reporting

## 🚀 Quick Commands

### Run Tests
```bash
# All tests with full reporting
python run-tests-with-report.py

# Smoke tests only
pytest -m smoke -v --log-cli=true

# Specific test file
pytest tests/test_smoke.py -v --log-cli=true

# With debug logging
pytest tests/ -v --log-cli-level=DEBUG

# With JSON report
pytest tests/ -v --json-report --json-report-file=test-results/report.json
```

### Generate Reports
```bash
# Generate comprehensive report from existing logs
python generate-comprehensive-report.py

# Generate with pytest-html
pytest tests/ --html=test-results/custom-report.html --self-contained-html
```

### View Logs
```bash
# View all tests combined log
cat test-results/logs/all_tests.log

# View pytest main log
cat test-results/logs/pytest.log

# View specific test log (with wildcard)
cat test-results/logs/test_smoke_admin_login_page_loads_*.log

# Search for errors
grep ERROR test-results/logs/*.log

# Follow log file in real-time
tail -f test-results/logs/pytest.log
```

## 📊 Report Locations

| Report | Location | Generated By |
|--------|----------|--------------|
| Comprehensive Report | `test-results/comprehensive-test-report.html` | `generate-comprehensive-report.py` |
| Pytest HTML Report | `test-results/detailed-report.html` | pytest-html plugin |
| Pytest Log | `test-results/logs/pytest.log` | pytest |
| All Tests Log | `test-results/logs/all_tests.log` | conftest.py root logger |
| Individual Test Logs | `test-results/logs/test_*.log` | LoggerConfig class |

## 🎯 Common Scenarios

### Scenario 1: Test Failed - Debug It
```bash
# 1. Run the specific failing test with debug logging
pytest tests/test_smoke.py::test_smoke_admin_logout -v --log-cli-level=DEBUG

# 2. Check the detailed log file
cat test-results/logs/test_smoke_admin_logout_*.log

# 3. Look for ERROR or WARNING messages
grep -E "ERROR|WARNING" test-results/logs/test_smoke_admin_logout_*.log
```

### Scenario 2: Generate Report from Existing Logs
```bash
# No need to rerun tests, just regenerate report
python generate-comprehensive-report.py

# Then open it
open test-results/comprehensive-test-report.html  # macOS
xdg-open test-results/comprehensive-test-report.html  # Linux
start test-results/comprehensive-test-report.html  # Windows
```

### Scenario 3: Run Tests for CI/CD
```bash
# Run tests and capture exit code
python run-tests-with-report.py
STATUS=$?

# Use exit code in your CI/CD pipeline
if [ $STATUS -eq 0 ]; then
    echo "Tests passed!"
    # Deploy or continue
else
    echo "Tests failed!"
    # Send alert or abort
fi
```

### Scenario 4: Run Subset of Tests
```bash
# Only smoke tests
pytest -m smoke -v

# Multiple markers
pytest -m "smoke and not critical" -v

# Specific file
pytest tests/test_login.py -v

# Specific function
pytest tests/test_smoke.py::test_smoke_admin_login_page_loads -v
```

## 📝 Log Levels

Use appropriate log levels in your tests:

```python
logger.debug("Variable value: x=5")          # Most detailed
logger.info("User logged in successfully")   # General info
logger.warning("Using deprecated API")      # Warning
logger.error("Failed to connect to server")  # Error occurred
logger.critical("Database connection lost")  # Critical error
```

## 🔧 Configuration

### To change log level in pytest.ini:
```ini
# Console logging level
--log-cli-level=DEBUG   # Show debug in console

# File logging level
--log-file-level=INFO   # Save info and above to file
```

### To change report filename:
```bash
# In command line
pytest tests/ --html=test-results/my-report.html

# Or edit pytest.ini
addopts = --html=test-results/my-report.html
```

### To add custom formatter:
Edit `utils/logger.py` and modify:
```python
detailed_formatter = logging.Formatter(
    'YOUR_FORMAT_HERE',
    datefmt='%Y-%m-%d %H:%M:%S'
)
```

## 📈 Log Format Reference

### File Log Format (Detailed)
```
2024-05-06 14:23:45 - test_smoke_admin_login_page_loads - INFO - [test_smoke.py:35] - test_smoke_admin_login_page_loads() - Test: Verify admin login page loads
```

Format breakdown:
- `2024-05-06 14:23:45` - Timestamp
- `test_smoke_admin_login_page_loads` - Logger name
- `INFO` - Log level
- `test_smoke.py:35` - File and line
- `test_smoke_admin_login_page_loads()` - Function name
- `Test: Verify...` - Message

### Console Log Format (Simple)
```
INFO - Test: Verify admin login page loads
```

## 🎓 Using Logger in Tests

### Basic Usage
```python
def test_my_feature(logger):
    logger.info("Starting test")
    # test code
    logger.info("Test completed")
```

### With HTML Report
```python
def test_login(page, logger):
    logger.info("Navigating to login page")
    page.goto("https://example.com/login")
    
    logger.info("Entering credentials")
    page.fill("input[name='user']", "admin")
    
    logger.info("Submitting form")
    page.click("button[type='submit']")
    
    logger.info("✅ Login test passed")
```

## 🐛 Troubleshooting

| Issue | Solution |
|-------|----------|
| No logs generated | Run tests first: `pytest tests/ -v` |
| Logger not available | Add `logger` parameter to test function |
| Old logs not clearing | They're timestamped, check `test-results/logs/` |
| Report is empty | Run `python generate-comprehensive-report.py` |
| Wrong log level showing | Check `pytest.ini` and `--log-cli-level` setting |
| Can't find a test log | Look for `test_TESTNAME_*.log` with timestamp |

## 💡 Tips & Tricks

1. **Quick test + report**
   ```bash
   python run-tests-with-report.py
   ```

2. **Follow logs in real-time**
   ```bash
   tail -f test-results/logs/pytest.log
   ```

3. **Search across all logs**
   ```bash
   grep -r "search_term" test-results/logs/
   ```

4. **Export logs for analysis**
   ```bash
   cp test-results/logs/*.log /path/to/archive/
   ```

5. **Compare test runs**
   ```bash
   diff test-results/logs/pytest_old.log test-results/logs/pytest.log
   ```
